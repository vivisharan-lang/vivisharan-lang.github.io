Cloud Network Monitoring API

What is this project?

A cloud-based backend API that simulates a telecom network monitoring system. It provides endpoints to check system health and network traffic.

What does it do?

  • Returns network status
  • Shows system health
  • Simulates traffic load

Live Demo

Open API →

Try:
/ → status
/health → health check
/traffic → traffic

How I built it

1. Backend API
Built using Python + Flask with endpoints for status, health, and traffic.

2. Containerization
Used Docker to package the application.

3. Cloud Deployment
Deployed using Google Cloud Run and Artifact Registry.

4. Debugging
Resolved IAM permission issues and managed service accounts.

What I learned

  • How cloud services run backend systems
  • How Docker containers work
  • How to deploy with Cloud Run
  • How to debug real cloud permission issues

Why this matters

This project reflects how telecom systems expose APIs for monitoring network health and traffic in modern cloud environments.
